    gitee创建hexo博客踩坑

    发表于 2019-07-16更新于 2019-07-16阅读次数

    最近做的项目代码是放在码云上管理的，因为没搭梯子，感觉速度比GitHub要快很多。于是想着把博客复制一份镜像到gitee试试。

    在这里记录一下遇到的一些坑：

    只有一个index.html可以访问

    问题：

    只有一个没有样式的首页可以访问，其他都是404。

    原因：

    新建仓库的名称是blog,这导致了我的博客地址带有后缀的路径。在配置_config.yml时，还是按照原来github那套配置来(网址无路径)，只改了url和repo两个链接，忘记修改root，导致了这个问题

    修复方法：

    修改_config.yml文件的root配置为’/blog’。

    live2d插件无法使用

    问题：

    博客里live2d插件消失，打开控制台，发现跨域报错。

    原因：

    这个问题很好找，看控制台发现live2d的资源请求协议是https。而我的博客是http协议的，所以被禁止访问了。

    解决:

    在gitee page 服务页面勾选强制使用https再更新就好了。
